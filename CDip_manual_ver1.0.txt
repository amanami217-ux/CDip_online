================================================================
 CDip ver 1.0 説明書
 — Diplomacy 標準ルールの要約と本アプリの操作方法 —
================================================================

本書は二部構成である。第一部で Diplomacy（Allan B. Calhamer, 1959）
の標準ルールを要約し、第二部で本アプリ（単一HTMLファイルの対面
プレイ用裁定ボード）の操作方法を説明する。本アプリの裁定エンジンは
標準ルールおよび DATC（Diplomacy Adjudicator Test Cases）に準拠する。


────────────────────────────────────────
 第一部  Diplomacy 標準ルールの要約
────────────────────────────────────────

■ 1. ゲームの目的と構成
・7カ国（イギリス・フランス・ドイツ・イタリア・オーストリア＝
  ハンガリー・ロシア・トルコ）で1901年の欧州を舞台に争う。
・盤面は75地域（海19・内陸14・沿岸42）から成り、うち34地域が
  補給都市（SC: Supply Center）である。
・自国の所有SCが18に達した国が単独勝利する。
・乱数要素はなく、全プレイヤーの命令は同時に公開・解決される。
  交渉（外交）は自由であり、約束に拘束力はない。

■ 2. ユニットと地形
・ユニットは陸軍（A: Army）と艦隊（F: Fleet）の2種で、強さは同一。
  1地域に置けるユニットは常に1つである。
・陸軍は陸地（内陸・沿岸）を移動する。艦隊は海域と沿岸地域を移動
  し、内陸には入れない。
・所有ユニット数は所有SC数を超えられない（調整フェーズで一致させる）。

□ 分割沿岸
  スペイン（spa: 北岸nc/南岸sc）、サンクトペテルブルク（stp: nc/sc）、
  ブルガリア（bul: 東岸ec/南岸sc）は沿岸が分かれており、艦隊は
  どちらの沿岸にいるかを常に区別する。艦隊の隣接は沿岸ごとに
  異なる（例: stp/nc は bar・nwy のみに接する）。陸軍は沿岸を
  区別しない。

□ 運河・海峡
  キール（kie）とコンスタンティノープル（con）は運河を持ち、
  艦隊は地域内を通過して両側の海域（kie: hel↔bal、con: aeg↔bla）
  へ抜けられる。沿岸の区別はない。デンマーク（den）はバルト海口の
  海峡地域で、den を挟む swe との艦隊隣接がある一方、bal と ska は
  直接隣接しない。ジブラルタル海峡は存在せず、spa と naf は隣接
  しない（mao 経由のみ）。

■ 3. ターン構成
  1年は春（Spring）と秋（Fall）の2シーズンから成り、各シーズンは
  次の順で進む。
   (1) 外交（交渉）
   (2) 命令の記入
   (3) 全命令の同時公開と解決（行軍フェーズ）
   (4) 退去ユニットの退却または解体（退却フェーズ）
  秋の退却フェーズ終了後にSC所有権を更新し、続けて調整フェーズ
  （Winter: 建設・解体）を行う。

■ 4. 命令の種類
・維持 H （例: A Par H）その場に留まる。命令を書かなかった
  ユニットも維持として扱われる。
・移動 M （例: A Par - Bur）隣接地域へ移動する。艦隊が分割沿岸
  地域へ入る場合は沿岸を指定する（例: F Mao - Spa/nc）。
・支援 S 隣接する地域への移動または維持を支援する
  （例: A Mar S A Par - Bur ／ A Mar S A Bur）。支援は「支援先の
  地域に自分が移動できること」が条件である。
・輸送 C 海域の艦隊が陸軍の海路移動を運ぶ
  （例: F Nth C A Lon - Nwy）。輸送する艦隊すべてが同じ輸送命令を
  書く必要がある。

■ 5. 解決の基本原則
・全命令は同時に解決される。移動の成否は戦力比較で決まる。
  戦力 = 1 +（切断されなかった有効な支援の数）
・攻撃側の戦力が防御側（維持側・競合する他の移動側）の戦力を
  上回った場合のみ移動が成功する。同数では動けない（防御側有利）。
・同一地域へ複数の移動が競合し最大戦力が並んだ場合、全員が
  失敗する（スタンドオフ）。スタンドオフが起きた地域は、その
  ターン中「係争地」となる。
・維持しているユニットが上回る戦力で攻撃されると退去
  （dislodge）となり、退却フェーズへ回される。
・自国ユニットを自国の攻撃で退去させることはできない。また、
  他国の攻撃に自国ユニットへの支援を与えても、その支援は自国
  ユニットの退去には寄与しない（自減の禁止）。

■ 6. 支援の切断
・支援中のユニットが攻撃を受けると支援は切断される。ただし、
  支援の宛先地域からの攻撃では切断されない（支援はその方向に
  構えているため）。
・支援中のユニットが退去させられた場合は、宛先からの攻撃で
  あっても支援は必ず切断される。

■ 7. ヘッドトゥヘッド
・隣接する2ユニットが互いの地域へ直接移動を命じた場合、正面
  衝突となり、戦力が上回った側だけが成功し、劣った側は退去する。
  同数なら両者とも動けない。
・一方または両方がコンボイ経由の移動である場合は正面衝突とは
  ならず、すれ違い（入れ替わり）が成立しうる。

■ 8. コンボイ
・陸軍は、出発地と到着地がともに沿岸地域で、間を海域の艦隊の
  連鎖で結べる場合に海路移動できる。
・輸送を構成する艦隊が1隻でも退去させられると、その経路は崩壊
  する。代替経路が残っていれば移動は継続する。
・輸送艦隊への攻撃と輸送される陸軍の攻撃が循環するパラドックス
  は Szykman 規則で解決する。すなわち、輸送経路が攻撃で妨害
  されたコンボイ移動は不成立とし、そのコンボイ陸軍の攻撃・
  支援切断効果はなかったものとして扱う。

■ 9. 退却フェーズ
・退去したユニットは、次の条件をすべて満たす地域へ退却できる。
   - 自ユニットが通常移動できる隣接地域である
   - 空白である
   - 攻撃してきたユニットの出発地でない（ただしコンボイで
     退去させられた場合は攻撃元へも退却できる）
   - このターンにスタンドオフが起きた係争地でない
・複数の退去ユニットが同じ地域へ退却した場合、全員が解体される。
・退却先がない場合、または退却を望まない場合は解体する。

■ 10. 所有権の更新と調整フェーズ
・秋の退却フェーズ終了時点でユニット（種類は問わない）が存在
  するSCは、そのユニットの国の所有となる。ユニットが去っても
  所有権は他国が占有するまで保持される。
・調整フェーズでは各国のユニット数をSC数に一致させる。
   - SC数 > ユニット数: 差の分だけ建設できる（任意）。建設は
     「空白で、自国が所有している、自国の本国SC」に限る。
     艦隊は沿岸の本国SCのみで、分割沿岸（stp）では沿岸を指定する。
   - SC数 < ユニット数: 差の分だけ自軍ユニットを選んで解体する
     （義務）。
・本国SCがすべて他国に占領されている間は建設できない。

■ 11. 勝利
・所有SCが18以上となった国が出た時点でゲームは終了し、その国の
  単独勝利となる。合意による引き分け終了も一般的である。


────────────────────────────────────────
 第二部  本アプリの操作方法
────────────────────────────────────────

本アプリは単一のHTMLファイルであり、ブラウザで開くだけで動作
する（インストール・通信不要）。対面プレイの盤面・命令記録・
裁定を1台の端末で行うことを想定している。

■ 1. タイトル画面
・New game          : 1901年春から新規に開始する。
・Resume last game  : ブラウザに自動保存された前回の対局を再開
                      する（保存がある場合のみ表示）。
・Load a saved file : 書き出したJSONファイルを読み込んで再開する。
・Credits           : 実装・地形データ・ルールのクレジットを表示する。

■ 2. 画面構成
・ヘッダ  : 現在のシーズン・年・フェーズ表示、表示モード切替
            （All orders ▾ など）、Undo（↩）、Menu（≡）。
・地図    : ドラッグでパン、ホイールまたはピンチでズーム。
            地域をタップして命令を入力する。SCは小円で示され、
            所有国の色で塗られる。
・右パネル: 入力済み命令の一覧。各行の削除ボタンで命令を取り
            消せる。
・下部    : フェーズの主ボタン（Resolve all orders 等）。

■ 3. 表示・入力モード（命令入力先の切替）
ヘッダのモード切替ボタンから選択画面を開く。
・All orders : 全7カ国の命令を1画面で入力する（GM運用・検討用）。
・Hidden     : 閲覧のみ。命令・矢印は表示されない。
・各国ボタン : その国のユニットだけを操作する単独モード。他国の
               命令は見えない。入力を終えた国には done ✓ が付く。
モードは解決前ならいつでも切り替えられる。選択画面の
「Resolve all orders」からそのまま解決に進むこともできる。

□ 対面プレイの推奨手順
  (1) Hidden にして交渉時間をとる
  (2) 各プレイヤーが順に自国の単独モードを選び、他者に見えない
      よう命令を入力して選択画面に戻る（done ✓ を確認）
  (3) 全員の入力後、Resolve all orders で一斉解決する

■ 4. 命令の入力（行軍フェーズ）
・自国ユニットをタップすると命令メニューが開く。
   - Hold    : 維持
   - Move    : 続けて行先の地域をタップする
   - Support : 支援対象のユニット→（移動支援なら）その行先を
               順にタップする。維持支援は対象ユニットの地域を
               再度タップする
   - Convoy  : （海域の艦隊のみ）輸送する陸軍→その行先を順に
               タップする
・艦隊が分割沿岸地域（spa/stp/bul）へ移動する場合、沿岸選択の
  モーダルが開く。到達可能な沿岸が1つだけの場合は自動で決まる。
・陸軍の移動で陸路とコンボイの両方が可能な場合（隣接する沿岸
  地域間）、「By land / By convoy」の選択モーダルが開く。
  非隣接地への移動は自動的にコンボイ扱いとなる。
・命令を書かなかったユニットは維持として解決される。
・命令の変更は再入力で上書き、取消は右パネルの削除ボタンで行う。

■ 5. 解決と結果
・Resolve all orders を押すと確認の後に全命令が同時解決される。
・結果画面では成功・失敗が矢印と色で地図上に表示され、ログに
  標準記法（例: A Par - Bur, F Nth C A Lon - Nwy）で記録される。
・過去の結果は Menu → View history でいつでも参照できる。

■ 6. 退却フェーズ
・退去ユニットがある場合のみ発生する。対象ユニットをタップし、
  退却可能な地域（強調表示）を選ぶか、Disband で解体する。
・退却可能地がないユニットは自動的に解体される。
・全員分を入力して Confirm retreats で確定する。

■ 7. 調整フェーズ（Winter）
・秋の退却後、SC所有権が更新されてから調整に入る。
・建設: 対象国のモードで空白の自国本国SCをタップし、
  Build an army / Build a fleet（stpは沿岸選択付き）を選ぶ。
  建設可能数はパネルに「Builds n/m」で表示される。建設は任意で
  あり、権利を放棄してもよい。
・解体: 自軍ユニットをタップして選択する。必要数（Disbands n/m）
  を満たすまで確定できない。
・Confirm adjustments で確定し、翌年の春へ進む。

■ 8. 履歴・保存・やり直し
・自動保存: 操作のたびにブラウザへ自動保存される。タブを閉じても
  Resume last game で再開できる（ブラウザのサイトデータを消去
  すると失われる）。
・Menu（≡）: View history（棋譜）、Save as JSON（ファイル書出）、
  Load JSON（読込）、Back to title。長期保存や端末間の移動には
  JSON書出を用いること。
・Undo（↩）: 直前の確定操作（解決・退却確定・調整確定など）の
  前まで盤面を巻き戻す。表示モードは変更されない。

■ 9. 勝利判定
・秋の所有権更新直後に18SC到達を判定し、達成国があれば勝利画面
  を表示する。

■ 10. 裁定の実装仕様（DATC対応の選択事項）
本エンジンは標準ルールの範囲内で以下の解釈を採用している。
・コンボイ意図は明示フラグ方式（DATC 4.A.3 選択c）。陸路と海路の
  両方が可能な移動は入力時の選択に従い、コンボイ指定の移動は
  経路が崩壊しても陸路に切り替わらない。ただし輸送艦隊が最初
  から1隻も存在しない場合は陸路として扱う（DATC 6.G.8）。
・コンボイが関与する対向移動はヘッドトゥヘッドとしない
  （DATC 4.A.5）。
・コンボイパラドックスは Szykman 規則（第一部 §8）で解決する。
・艦隊の分割沿岸への移動で沿岸が曖昧な命令は無効（維持扱い）と
  する。通常はUIの沿岸選択により曖昧な命令は発行されない。

■ 11. クレジット
・実装            : Claude（Anthropic 開発のAIアシスタント）による
                    生成・検証
・地形データ      : Natural Earth（パブリックドメイン,
                    naturalearthdata.com）由来の海岸線・地形
・ルール          : Diplomacy 標準ルール（Allan B. Calhamer, 1959）
・裁定準拠        : DATC — Diplomacy Adjudicator Test Cases
                    （Lucas B. Kruijswijk）
・本アプリは対面プレイ補助を目的とした非公式のファンメイド
  ツールである。Diplomacy の商標権は権利者に帰属する。
